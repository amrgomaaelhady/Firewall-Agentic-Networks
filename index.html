The Research

Two Companion Papers

ConVerse reveals the problem: AI assistants systematically leak private data and fall for social engineering in multi-agent settings. The Firewalls paper provides the solution: a dual-firewall architecture that structurally eliminates both attack vectors while preserving task utility.

🛡️ Firewalls: The Defense

A dual-firewall architecture providing structural protection. The Data Abstraction Firewall controls outgoing data granularity; the Language Converter Firewall eliminates natural language manipulation by converting inputs to a closed structured protocol.

How It Works

The Dual-Firewall Architecture

Data Abstraction Firewall (outgoing path): When the assistant queries user data, responses pass through the firewall which applies learned rules to filter, abstract, or pass information according to contextual appropriateness. Example: a home address becomes "departing from the Paris area."

Language Converter Firewall (incoming path): Messages from external agents are converted to a closed, structured JSON protocol with validated fields. Persuasive framing, urgency tactics, and social engineering are structurally eliminated — the attack surface of arbitrary natural language is removed entirely.

Both firewalls operate in a trusted environment isolated from external input, applying domain-specific rules learned automatically from demonstrations. Protection holds regardless of attack sophistication.
